# Организация правил css

Выработанный опытом порядок организации каскада внутри правил препроцессорного css-кода, позволяющий значительно упростить и ускорить верстку, а также поддерживать уже
существующий код через какое-то время после окончания активных работ.

Порядок такой. Сначала – сущности, которые вложены в родительский селектор, потом – которые модифицируют его (или вложенные в него псевдоэлементы и теги, тогда используется тот же порядок).

* псевдоэлементы (никакой внутренней вложенности),
* вложенные теги и классы (на вложенность самого родительского селектора никак не влияет),
* псевдоклассы (в которых может быть переопределение как псевдоэлементов, так и вложенных тегов),
* модификаторы (в которых может быть как переопределение вложенных структур само по себе, так и за псевдоклассами, причем и то и другое может иметь место, тогда внутренний порядок – по тем же принципам).

Пример написан на LESS, но вышеописанное справедливо и для SCSS, и для Sass, и для Stylus.
Редко когда будут задействованы все показанные правила, при этом чаще всего описание отдельной сущности не превысит один экран, и не придется туда-сюда перемещаться по файлу.
Оптимизация количества медиазапросов производится [средствами gulp](https://www.npmjs.com/package/gulp-group-css-media-queries).

```less
@tablet: ~"(min-width: 768px)";

.block {
  margin: 0;

  @media @tablet {
    margin: 0;
  }

  &::before {
    margin: 0;

    @media @tablet {
      margin: 0;
    }
  }
  
  // стараюсь не использовать вложенные теги, но если приходится, то таким способом и в этом месте
  a {
    margin: 0;
    
    @media @tablet {
      margin: 0;
    }

    &::before {
      margin: 0;

      @media @tablet {
        margin: 0;
      }
    }

    &:hover {
      // ...
    }
  }
  
  .active {
    // ...
  }

  &:hover {
    margin: 0;

    @media @tablet {
      margin: 0;
    }

    &::before {
      // ...
    }
  
    a {
      // ...
    }
  
    .active {
      // ...
    }
  }

  &--modifier {
    margin: 0;

    @media @tablet {
      margin: 0;
    }

    &::before {
      // ...
    }
  
    a {
      // ...
    }
  
    .active {
      // ...
    }

    &:hover {
      margin: 0;

      @media @tablet {
        margin: 0;
      }

      &::before {
        // ...
      }

      a {
        // ...
      }

      .active {
        // ...
      }
    }
  }
}

.block__element {
  // ...
}
```
